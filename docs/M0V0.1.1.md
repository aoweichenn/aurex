# Aurex M0V0.1.1

M0V0.1.1 is the first engineering baseline for the Aurex bootstrap compiler.
It keeps the production compiler modular while adding a standalone bootstrap
compiler, regression tests, and a small performance smoke benchmark.

## Layout

- `include/aurex/base`: source management, diagnostics, result type, integer aliases.
- `include/aurex/syntax`: tokens, AST IDs, compact AST storage, AST dump.
- `include/aurex/lex`: handwritten lexer.
- `include/aurex/parse`: handwritten recursive descent parser over `std::span<const Token>`.
- `include/aurex/sema`: type table, symbol table, semantic analyzer.
- `include/aurex/codegen_c`: C backend.
- `include/aurex/driver`: file IO and compile pipeline.
- `src/cli`: argument parsing only.
- `bootstrap`: standalone single-file bootstrap compiler with its own Makefile.
- `tools`: test and benchmark entry points.

## Current Language Coverage

Supported syntax includes module/import, extern c, export c fn, opaque struct,
struct, enum, const, fn, let/var, if/else, while, break/continue, return,
pointers, arrays, string/C string/byte/integer/bool/null literals, calls,
field/index access, struct literals, and explicit cast builtins.

M0V0.1.1 sema checks the most important cost-transparency rules: no function
overloading, no shadowing, explicit return matching, bool-only conditions,
array restrictions on parameters/returns/assignments, no assignment to `let`,
opaque structs only behind pointers, and exact type matching except integer
literals into integer destinations.

## Build

```sh
cmake -S . -B build
cmake --build build -j
```

## Test

```sh
tools/run_tests.sh
```

## Benchmark

```sh
tools/bench.py
```

## Bootstrap Compiler

```sh
make -C bootstrap
bootstrap/m0_bootstrap examples/hello.ax -o bootstrap/hello.bootstrap.c
cc bootstrap/hello.bootstrap.c -o bootstrap/hello.bootstrap
bootstrap/hello.bootstrap
```

The bootstrap compiler is deliberately small and heavily commented. It is a
readable Stage0 seed, not a replacement for the modular compiler.
