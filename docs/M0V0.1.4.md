# Aurex M0V0.1.4

M0V0.1.4 moves the project closer to a measurable self-hosting path.

## Added

- Root `Makefile` with `build`, `test`, `bench`, and `bootstrap-chain` targets.
- `--dump-checked` and `--emit=checked` CLI mode for semantic side-table
  summaries.
- Golden token test script: `tools/check_golden.sh`.
- Golden token output for `examples/hello.ax`.
- Expanded `selfhost/src/lexer_smoke.ax` from token counting to token kind
  sequence validation.

## Improved

- Selfhost bootstrap chain now verifies:
  - `m0c_seed.ax`;
  - `lexer_smoke.ax`;
  - standalone bootstrap compiler output.
- Parser condition parsing no longer confuses `if a != b { ... }` with a struct
  literal.

## Verification

```sh
make -C aurex test
make -C aurex bootstrap-chain
make -C aurex bench
```

The project is still not fully self-hosted. The current selfhost lexer smoke is
the first concrete M0 compiler-component seed; the next step is turning it into
a token dump generator that can match the C++ lexer on golden files.
