# Aurex M0V0.1.5

M0V0.1.5 turns the selfhost lexer track from a pure pass/fail smoke test into a
measurable token dump.

## Added

- `selfhost/src/lexer_dump.ax`: an M0 program that scans an embedded M0 source
  string and prints one token kind per line.
- `selfhost/src/lexer_file.ax`: an M0 lexer driver that reads a source file
  through explicit runtime file IO and prints token kinds.
- `tests/golden/selfhost_lexer_dump.tokens`: golden output for the M0 lexer
  dump.
- `tests/golden/selfhost_lexer_file_hello.tokens`: golden output for scanning
  `examples/hello.ax` through the file-backed M0 lexer driver.
- Bootstrap-chain verification for the M0 lexer dump.
- `selfhost/Makefile` with a local `check` target for all selfhost seeds.
- `tools/compare_selfhost_lexer.sh`: direct comparison between the M0
  file-backed lexer and the production C++ Stage0 lexer on `examples/hello.ax`
  plus every local positive/negative test input.
- C++ Stage0 module loading for `import a.b;`, resolving `a/b.ax` from the
  importer directory or `-I` search roots.
- `tests/positive/module_math.ax` and `tests/positive/import_path.ax` module
  loading coverage.
- Source-ranged diagnostics for missing imports and module-name mismatches.
- Cyclic import detection.
- `--dump-modules` / `--emit=modules` for inspecting resolved module names and
  paths.
- Positive regression tests:
  - `tests/positive/condition_regression.ax` covers `if actual != expected {}`.
  - `tests/positive/pointer_ops.ax` covers address-of, dereference, and pointer
    mutation codegen.

## Improved

- `tools/check_golden.sh` now validates both the C++ lexer hello token dump and
  both M0 selfhost lexer dumps.
- `tools/run_tests.sh` checks `lexer_dump.ax`, `lexer_file.ax`, and runs behavior-sensitive
  positive examples that should return zero.
- `bootstrap_chain.sh` now proves the M0 lexer agrees with Stage0 on token
  kinds for the local corpus.
- `m0_driver` now owns import resolution and AST merging; parser, sema, and
  codegen remain independent of file loading.
- Imported modules are validated against the path used to import them.
- Documentation now describes the current selfhost boundary and the new
  file-backed lexer driver.
