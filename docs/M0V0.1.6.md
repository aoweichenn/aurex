# Aurex M0V0.1.6

M0V0.1.6 makes the selfhost lexer track use real M0 modules. This is still not
full self-hosting, but the selfhost code now depends on the Stage0 module loader
instead of duplicating scanner logic in each entry point.

## Added

- `selfhost/src/aurex/selfhost/lexer/core.ax`: shared M0 lexer core with token
  constants, character predicates, keyword matching, trivia skipping, literal
  scanning, and `scan_next`.
- `selfhost/src/aurex/selfhost/lexer/dump.ax`: shared M0 token-kind dump helper
  built on the core scanner.
- Selfhost module regression checks in `selfhost/Makefile`,
  `tools/bootstrap_chain.sh`, and `tools/run_tests.sh`.
- Documentation for manually compiling selfhost sources with
  `-I selfhost/src`.

## Improved

- `selfhost/src/lexer_smoke.ax` now imports the shared scanner core.
- `selfhost/src/lexer_dump.ax` now imports the shared dump module and only owns
  the embedded input source.
- `selfhost/src/lexer_file.ax` now imports the shared dump module and only owns
  file IO plus entry-point behavior.
- `tools/check_golden.sh`, `tools/compare_selfhost_lexer.sh`, and
  `tools/bootstrap_chain.sh` pass the selfhost import root consistently.
- The selfhost tests assert that `lexer_file.ax` loads both
  `aurex.selfhost.lexer.dump` and `aurex.selfhost.lexer.core`.

## Verification

```sh
cmake --build build -j
tools/run_tests.sh
tools/check_golden.sh
tools/bootstrap_chain.sh
make -C selfhost check
tools/compare_selfhost_lexer.sh
tools/bench.py
```

The next selfhost milestone is to make the M0 lexer return token ranges and
source text, then feed that token stream into a small M0 parser seed.
