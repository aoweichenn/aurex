# Aurex M0V0.1.7

M0V0.1.7 moves the selfhost lexer one step closer to parser input. The M0
scanner now returns token kind plus byte ranges, while existing token-kind dump
tests remain compatible.

## Added

- `TokenSpan` in `selfhost/src/aurex/selfhost/lexer/core.ax`.
- `scan_token(source, index) -> TokenSpan`, returning `kind`, `begin`, and
  `end` byte offsets.
- `scan_next` is retained as a compatibility wrapper over `scan_token`.
- `selfhost/src/lexer_ranges.ax`, a range-aware selfhost lexer smoke test.
- Range validation in `selfhost/Makefile`, `tools/bootstrap_chain.sh`, and
  `tools/check_golden.sh`.

## Improved

- The selfhost lexer core is now useful to a future parser seed, not only to
  token-kind dump programs.
- The bootstrap chain now verifies:
  - selfhost seed execution;
  - token sequence scanning;
  - token range scanning;
  - embedded token dump golden output;
  - file-backed token dump golden output;
  - C++ Stage0 lexer vs M0 selfhost lexer corpus agreement.

## Verification

cmake --build build -j
tools/run_tests.sh
tools/check_golden.sh

then start a minimal M0 parser seed.
