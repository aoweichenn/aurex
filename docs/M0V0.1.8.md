# Aurex M0V0.1.8

M0V0.1.8 adds the first parser seed written in M0. It is intentionally small:
it validates a fixed module/import/extern/function-signature source and now
produces an ID-backed AST node graph for that covered syntax. The point is to
prove the selfhost lexer `TokenSpan` stream can drive parser code and syntax
data output written in M0. The parser seed is now split by role, and the
expression path uses an iterative operator/frame-stack parser instead of
recursive Pratt parsing.

## Added

- `selfhost/src/aurex/selfhost/parser/cursor.ax`: shared one-token cursor
  helpers.
- `selfhost/src/aurex/selfhost/parser/types.ax`: type/signature parser using an
  iterative pointer-prefix stack.
- `selfhost/src/aurex/selfhost/parser/expr.ax`: expression parser using
  explicit operator and frame stacks.
- `selfhost/src/aurex/selfhost/parser/seed.ax`: module/item/block orchestration
  for the parser seed.
- `selfhost/src/aurex/selfhost/syntax/ast.ax`: ID-backed node-pool AST data
  module for the parser seed subset.
- `selfhost/src/aurex/selfhost/smoke/parser_smoke.ax`: executable parser smoke
  test.
- Parser seed checks in `selfhost/Makefile`, `tools/bootstrap_chain.sh`,
  `tools/check_golden.sh`, and `tools/run_tests.sh`.
- Module-load regression checks for `aurex.selfhost.parser.seed`,
  `aurex.selfhost.parser.cursor`, `aurex.selfhost.parser.expr`,
  `aurex.selfhost.parser.types`, `aurex.selfhost.syntax.ast`, and
  `aurex.selfhost.lexer.core`.

## Parser Seed Coverage

The seed parser currently covers:

- `module a.b;`
- `import a.b;`
- `extern c { fn ... @name("..."); }`
- `extern c { opaque struct ...; }`
- primitive and pointer types used in function signatures
- `export c fn ... { ... }` with block, expression statement, return statement,
  call expression, full call-argument expressions, literal/identifier
  expression nodes, unary operators, and iterative operator-stack binary
  expression precedence

It does not yet build the full production AST, report rich diagnostics, or parse
the full statement/expression language structurally.

subset, then compare a stable M0 parse dump with the C++ parser on a small
shared corpus.
