# Aurex M0V0.1.8

M0V0.1.8 adds the first parser seed written in M0. It is intentionally small:
it validates a fixed module/import/extern/function-signature source rather than
building a full AST. The point is to prove the selfhost lexer `TokenSpan` stream
can drive recursive-descent parser code written in M0.

## Added

- `selfhost/src/aurex/selfhost/parser/seed.ax`: shared parser seed module.
- `selfhost/src/parser_smoke.ax`: executable parser smoke test.
- Parser seed checks in `selfhost/Makefile`, `tools/bootstrap_chain.sh`,
  `tools/check_golden.sh`, and `tools/run_tests.sh`.
- Module-load regression checks for `aurex.selfhost.parser.seed` and
  `aurex.selfhost.lexer.core`.

## Parser Seed Coverage

The seed parser currently covers:

- `module a.b;`
- `import a.b;`
- `extern c { fn ... @name("..."); }`
- primitive and pointer types used in function signatures
- `export c fn ... { ... }` with a minimal brace-balanced body skip

It does not yet build an AST, report rich diagnostics, or parse statements and
expressions structurally.
