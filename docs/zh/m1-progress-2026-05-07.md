# M1 进度报告

日期：2026-05-07

范围：M1 验收样例、标准库 OS 能力、subprocess stdout/stderr capture + cwd/env baseline、file metadata / mtime baseline、directory create / directory entries / recursive directory entries / single-level source discovery / recursive source discovery baseline、Map / CStringUsizeMap baseline、target graph validation / topological build baseline、target name lookup cache baseline、target graph diagnostic/message/name/cycle-path/cycle-path-name baseline、测试支撑 direct process runner、集成测试覆盖、测试耗时基线。
结论：M1 还没有完整完成，但已经从路线图推进到 active tree 中的可运行验收基线。当前项目已经能用 Aurex 写出两个最小系统级样例：一个 compiler frontend slice，以及一个 typed build-tool slice；两者都有 checked dump、IR dump 和 native run 覆盖。typed build-tool 已经开始使用 captured stdout/stderr、cwd/env、真实 source/stamp mtime 比较、目录创建、拥有型目录项读取、单层目录 source discovery 计数、递归目录 source discovery 计数、递归目录 entry source discovery、target name lookup cache、target graph 拓扑构建顺序、结构化 graph diagnostic、稳定 message、project-aware target/related name accessors、cycle path index baseline 和 cycle path name baseline，而不只是检查子进程 exit code 或 stamp 是否存在。测试支撑层也已经避免简单 native smoke 再通过 `/bin/sh -c`，可直接 fork/exec 并捕获 stdout/stderr；新增嵌套目录验收也不再通过 `/bin/mkdir` 子进程建目录。

2026-05-08 增量：字符串基础类型风险收口继续推进。`std.fs.dir` 已从“目录项内部 UTF-8 `String`”改为 bytes-backed `Path`，公开层不再暴露 unchecked `DirectoryEntry.name()/path() -> str`；新 API 改为 `name_bytes()` / `path_bytes()` 原始 bytes 视图和 `name_utf8()` / `path_utf8()` checked UTF-8 视图。目录 suffix 新增 `str` 包装并拒绝内部 NUL。M1 axbuild 的目录扫描已迁到 `Path` + `str` suffix + bytes entry 名称匹配，临时 source 准备也改用 `create_directory_path` 和 `write_str_path`，减少业务代码直接传 `c"..."` 的范围。

## 1. 本轮目标

这轮工作的目标不是继续补零散语法点，而是验证 M1 的核心问题：

- Aurex 是否已经能自然表达编译器前端中的 source manager、diagnostic、lexer、token stream 和 parser 子集。
- Aurex 是否已经能自然表达 typed build graph、target、dependency、custom command、incremental check、build、clean、run 和 test 流程。
- 构建工具能否使用 typed argv subprocess，而不是继续依赖 shell 字符串拼接。
- 新增能力是否能进入 integration tests，避免只靠手动 smoke。
- 新增验收用例是否会明显拖慢已有测试入口。

## 2. 已落地内容

### 2.1 标准库 subprocess stdout/stderr capture + cwd/env baseline

新增 `std.sys.process::Command`，作为 M1 build-tool 的 typed subprocess 基线：

- 文件：`std/sys/process.ax`
- 新增结构：`Command`
- 新增 API：`command()`、`command_arg()`、`command_run()`、`command_run_capture()`、`command_destroy()`
- 新增 cwd/env API：`command_cwd()`、`command_env()`
- 新增 method API：`Command.arg()`、`Command.cwd()`、`Command.env()`、`Command.run()`、`Command.run_capture()`、`Command.destroy()`
- 新增拥有型输出：`ProcessOutput`
- 新增输出 API：`ProcessOutput.exit_status()`、`ProcessOutput.stdout()`、`ProcessOutput.stdout_c_data()`、`ProcessOutput.stderr()`、`ProcessOutput.stderr_c_data()`、`ProcessOutput.destroy()`
- 返回模型：`Result<i32, i32>`，成功时返回子进程 exit status。

底层 host-c support 新增：

- 文件：`std/ffi/c/support/host_c.c`
- 新增 ABI：`aurex_std_v0_run_process`
- 新增 ABI：`aurex_std_v0_run_process_capture`
- 新增 ABI：`aurex_std_v0_run_process_with_options`
- 新增 ABI：`aurex_std_v0_run_process_capture_with_options`
- 新增 ABI：`aurex_std_v0_free_process_output_data`
- 实现方式：`fork` / `execvp` / `waitpid`
- stdout/stderr capture 通过独立 pipe + `dup2(STDOUT_FILENO)` / `dup2(STDERR_FILENO)` 实现，父进程用 `poll` 同时 drain 两个 pipe，避免只读单路输出导致子进程在另一条输出上阻塞；返回的 buffer 由 `ProcessOutput.destroy()` 释放。
- cwd/env 通过 options ABI 在 child `execvp` 前应用，`Command.env()` 当前接收 `KEY=VALUE` 形式的 assignment。
- capture FFI 使用调用方传 `HostProcessOutput*` 的形状，避免大 struct by-value return 在 arm64 C ABI 上出现不稳定约定。
- 失败约定：
  - program 为空、参数非法：返回 `127`
  - fork、malloc、waitpid 等 host failure：返回 `126`
  - 子进程被 signal 终止：返回 `128 + signal`
- `waitpid` 已处理 `EINTR`，避免信号打断导致误报失败。

当前还未完成：

- stdin/stdout/stderr pipe。
- timeout / kill。
- 更结构化的 process error enum。

### 2.3 标准库 directory create + entries + source discovery baseline

新增 `std.fs.dir`，作为 M1 build-tool source discovery 的低风险目录扫描基线：

- 文件：`std/fs/dir.ax`
- 新增 API：`create_directory(path) -> bool`
- 新增 Path API：`create_directory_path(path) -> bool`
- 新增结构：`DirectoryEntry`
- 新增 enum：`DirectoryEntryKind`
- 新增 API：`read_entries(path) -> Result<Vec<DirectoryEntry>, i32>`
- 新增 API：`read_entries_recursive(path) -> Result<Vec<DirectoryEntry>, i32>`
- 新增 Path API：`read_entries_path(path) -> Result<Vec<DirectoryEntry>, i32>`
- 新增 Path API：`read_entries_recursive_path(path) -> Result<Vec<DirectoryEntry>, i32>`
- 新增 API：`destroy_entries(entries)`
- 新增 entry API：`DirectoryEntry.name_c_data()`、`DirectoryEntry.path_c_data()`、`DirectoryEntry.name_bytes()`、`DirectoryEntry.path_bytes()`、`DirectoryEntry.name_utf8()`、`DirectoryEntry.path_utf8()`、`DirectoryEntry.kind()`、`DirectoryEntry.is_file()`、`DirectoryEntry.is_directory()`、`DirectoryEntry.is_symlink()`
- 新增 API：`count_files_with_suffix(path, suffix) -> Result<i32, i32>`
- 新增 API：`has_file_with_suffix(path, suffix) -> Result<bool, i32>`
- 新增 API：`count_files_with_suffix_recursive(path, suffix) -> Result<i32, i32>`
- 新增 API：`has_file_with_suffix_recursive(path, suffix) -> Result<bool, i32>`
- 新增 suffix `str` API：`count_files_with_suffix_str`、`count_files_with_suffix_path_str`、`has_file_with_suffix_str`、`has_file_with_suffix_path_str` 及递归版本，内部通过 `CString.from_str` 拒绝内部 NUL。
- 返回模型：目录创建当前返回 `bool`；目录项读取、计数和存在性查询使用 `Result`，调用方可以继续用 `?` 串联错误传播。
- 目录项所有权：host-c 返回的 name/path 会在 `std.fs.dir` 中复制成 bytes-backed `Path`，随后释放 host buffer；调用方用完 `Vec<DirectoryEntry>` 后必须调用 `destroy_entries()` 释放 entry 内部路径 buffer。目录项默认按 path bytes 处理，只有调用 `name_utf8()` / `path_utf8()` 时才显式进入 UTF-8 文本视图。

底层 host-c support 新增：

- 文件：`std/ffi/c/support/host_c.c`
- 新增 ABI：`aurex_std_v0_directory_create`
- 新增 ABI：`aurex_std_v0_directory_read_entries`
- 新增 ABI：`aurex_std_v0_directory_read_entries_recursive`
- 新增 ABI：`aurex_std_v0_directory_free_entries`
- 新增 ABI：`aurex_std_v0_directory_count_files_with_suffix`
- 新增 ABI：`aurex_std_v0_directory_count_files_with_suffix_recursive`
- 实现方式：POSIX `opendir` / `readdir` / `closedir` + `stat`
- 单层统计只统计当前目录普通文件，并按文件名后缀过滤。
- `read_entries` 返回当前目录下非 `.` / `..` entry 的 name、完整 path 和 kind，kind 通过 `lstat` 区分 file / directory / symlink / other。
- `read_entries_recursive` 返回根目录以下的递归 `Vec<DirectoryEntry>`，包含子目录 entry 和文件 entry；递归下降只进入真实目录，不跟随 symlink 目录，避免符号链接环。
- 递归统计会进入真实目录，统计普通文件；递归路径使用 `lstat` 判断目录，当前不跟随 symlink 目录，避免符号链接环。
- FFI 形状：调用方传 `i32*` 输出参数，继续避开 struct by-value return ABI 风险。

当前还未完成：

- streaming directory iterator / walk callback。
- glob / pattern。
- 更完整 symlink 策略。
- 临时目录创建/销毁 API。
- path normalization。

### 2.4 标准库 Map / CStringUsizeMap baseline

`std.core.map` 已有泛型 `Map<K, V>` 的 Vec-backed baseline，适合当前语言阶段的简单 key/value 容器验收；本轮在不改变泛型 `Map<K, V>` 行为的前提下，新增 borrowed C string key 到 `usize` 的专用 lookup baseline：

- 文件：`std/core/map.ax`
- 新增结构：`CStringUsizeEntry`
- 新增结构：`CStringUsizeMap`
- 新增 API：`cstring_usize_new()`、`cstring_usize_with_capacity()`、`cstring_usize_destroy()`
- 新增 API：`cstring_usize_len()`、`cstring_usize_capacity()`、`cstring_usize_is_empty()`、`cstring_usize_reserve()`、`cstring_usize_clear()`
- 新增 API：`cstring_usize_find_index()`、`cstring_usize_contains()`、`cstring_usize_insert()`、`cstring_usize_insert_absent()`、`cstring_usize_set()`、`cstring_usize_get()`、`cstring_usize_remove()`
- 新增 method API：`CStringUsizeMap.with_capacity()`、`insert()`、`insert_absent()`、`set()`、`get()`、`remove()`、`contains()`、`find_index()`、`clear()`、`destroy()`
- key 模型：当前 key 是 borrowed null-terminated C string，不拥有字符串内存；比较使用 `text::c_string_equal()`，避免 `*const u8` 泛型 key 只按指针地址比较。
- 当前用途：`examples/m1/axbuild` 的 `Project` 已维护 `target_lookup: CStringUsizeMap`，`Project.find_target()` 走 lookup cache，同时保留 `project_find_target_linear()` 作为对照验收，确保 cache 结果和旧线性扫描一致。

当前还未完成：

- hash map / bucketed map。
- owned `String` key map。
- 可自定义 equality / hash 的泛型 Map。
- remove 后 tombstone / rehash 等更真实性能策略。

## 5. 当前 M1 完整度判断

可以认为已经完成：

- M1 两个验收程序的最小 active-tree baseline。
- 测试支撑层的 `aurexc` in-process fast path、编译结果缓存、native artifact 缓存和简单命令 direct fork/exec runner。
- typed subprocess 的最小标准库入口。
- stdout/stderr capture 和 cwd/env 的最小标准库入口。
- file metadata / mtime 的最小标准库入口。
- directory create、拥有型单层/递归目录项读取、单层 source discovery 和递归 source discovery 计数的最小标准库入口。
- frontend slice 的 source manager / diagnostics / lexer / token stream / parser summary。
- build-tool slice 的 project / target / dependency / target name lookup cache / duplicate target detection / graph diagnostic/message/name/cycle index path/cycle name path / command / captured stdout/stderr / cwd/env / source-stamp mtime incremental / directory create / directory entry source discovery / recursive directory entry source discovery / single-level and recursive directory source discovery / target graph validation / topological build order / run / test 主流程。
- checked / IR / native 三层测试保护。
- 文档和 examples README 的入口说明。

不能认为已经完成：

- 完整自举前端。
- 完整 typed build tool。
- 完整 OS/process 标准库。
- 完整资源管理模型。
- 完整泛型约束、trait、class/object model。
- 完整 pattern matching 工业级诊断。
- 完整 AIR 后端契约。

一句话结论：

M1 已经从“规划中的功能集合”推进到了“可运行、可测试、可继续扩展的验收骨架”。下一步不是重新开路线，而是在这个骨架上继续补真实工程能力。

## 6. 下一步建议

优先级建议如下：

1. 继续增强 `std.sys.process::Command`
   - 加 stdin/stdout/stderr pipe。
   - 加 timeout。
   - 把状态码升级为更结构化的 process result。

2. 补目录和更完整 metadata API
   - 当前已有最小 metadata、mtime、is_file、is_dir、目录创建、拥有型单层/递归目录项读取，以及按后缀统计普通文件的单层/递归目录扫描入口。
   - 下一步补 streaming directory iterator / walk callback、glob/pattern、lstat/symlink metadata、权限/mode 和 path normalization。

3. 继续完善 `Map<K, V>` / string-key map
   - 当前已有 Vec-backed 泛型 `Map<K, V>` 和 borrowed C string -> usize lookup baseline。
   - axbuild 已用 `CStringUsizeMap` 维护 target name -> id lookup cache。
   - 下一步补 owned `String` key、hash/bucket、可自定义 equality/hash，再服务 frontend symbol table。

4. 扩展 frontend 样例
   - 从 summary parser 推进到 AST node list。
   - 加 source span diagnostic。
   - 加 parser error recovery。
   - 加 module/import token 解析。

5. 扩展 axbuild 样例
   - 带 dependency value 的 graph diagnostic report。
   - 真实 compile/link command modeling。
   - build/test 命令 stderr 捕获和结构化报告。

6. 保持测试性能纪律
   - checked/IR 断言覆盖大部分 feature surface。
   - native run 只保留关键端到端 smoke。
   - 简单 native smoke 走 direct fork/exec，避免 `/bin/sh -c` 干扰。
   - 大样例进入慢测分组前必须有明确价值。
   - 每轮新增测试后记录 focused test 和 `tools/run_tests.sh` 的 real time。
