# Aurex M3.5 Incremental Syntax And Stable AST Identity 计划

## 状态

M3.5 已在 `m3.5` 分支完成当前确定性 tooling/syntax 边界。

本阶段不新增语言语法，也不引入第二套 parser。它把 M3.4 已经打通的 incremental sema execution 下沉到
lossless syntax 和 AST projection 层，让 IDE 反复小编辑时可以用稳定身份解释哪些 syntax subtree、AST item/body
和 semantic fact 仍然对齐。

## 完成交付

1. **range-based document edit**
   - `ToolingDocumentTextEdit` 描述 `begin`、`removed_length` 和 `inserted_text`。
   - `ToolingSession::change_document_range(...)` 支持 versioned document store 上的小范围编辑。
   - `ToolingSession::change_document_range_with_reuse_plan(...)` 在同一个入口返回版本、实际 edit、精确
     `IdeEditImpact`、reuse plan 和 incremental snapshot result。
   - 越界 edit 返回 `invalid_source`，不会破坏当前 open document state。

2. **position-independent lossless syntax stable key**
   - `LosslessNodeStableKey` 不使用绝对 source range 或 token index。
   - key 由 node kind、identifier anchor hash、subtree token fingerprint、token/semantic-token/child shape 和 depth
     组成。
   - 局部 prefix edit 后，未改变的 subtree 可以保持同一个 stable key；旧的 `LosslessNodeKey` 继续保留 range/token
     形状，服务精确定位和 debug。

3. **lossless subtree reuse stats**
   - `compare_lossless_stable_nodes(previous, current)` 以 stable key multiset 比较两棵 immutable lossless tree。
   - `LosslessSyntaxReuseStats` 报告 previous/current nodes、reused、recomputed、invalidated 和 stable-key
     collision 计数。
   - `ToolingIncrementalSnapshotResult::syntax_reuse` 将这些计数暴露给 tooling/session 调用方。

4. **stable AST item/body projection**
   - `IdeAstNodeInfo` / `ToolingAstNode` 提供 offset 到 AST item 或 function body 的 protocol-neutral projection。
   - item projection 暴露稳定 `DefKey` 字符串；function body projection 暴露稳定 `DefKey` + `BodyKey` 字符串。
   - 这让 offset-to-token、syntax-node、AST-node 和 semantic-fact projection 能在同一个 snapshot/result 中对齐。

5. **正式测试**
   - `LosslessStableNodeKeySurvivesPrefixEdit` 验证 prefix edit 后未变函数 subtree 的 stable key 保持一致。
   - `ToolingSessionRangeEditReportsSyntaxReuseAndStableAstBody` 验证 range edit、syntax reuse stats 和 AST body stable
     key。
   - `ToolingSessionAppliesRangeEditWithoutReusePlan` 验证非 reuse-plan 的 range edit 入口也会产生可复用 previous
     context。

## 明确边界

- 当前 M3.5 仍然保持 eager aggregate snapshot 构建：lexer、lossless tree、AST lowering 和 sema aggregate 仍会构建当前
  snapshot。
- M3.5 的“syntax reuse”是基于 stable key 的 immutable subtree reuse 解释和 tooling result，不把旧节点对象物理 graft
  到新树里。
- 完整 subtree reparse、green-tree 节点物理共享、background scheduler 和持久 project DB 继续后置。
- M3.6 才处理 project graph 与 persistent query DB；M3.7 才在稳定 snapshot/reuse 基础上新增 completion、rename、
  semantic tokens 等 IDE 高层能力。

## 验收

- range edit 和 full replacement 能共存，且都保持 stale version 检查。
- 未变化 lossless subtree 在前缀/局部编辑后能通过 `LosslessNodeStableKey` 重新匹配。
- `ToolingIncrementalSnapshotResult` 能同时报告 sema query reuse、syntax reuse 和 workspace-index update。
- AST item/body projection 的稳定 key 与 semantic facts 使用的 `DefKey`/`BodyKey` 形状一致。
- 现有构建、gtest、coverage、query pruning、query graph fuzz 和 generic stress gates 必须保持 green。
