# Aurex M3.6 Project Graph And Persistent Query DB 计划

状态：2026-05-30，M3.6 已在 `m3.6` 分支完成当前工程图和持久 query DB 边界。

## 阶段定位

M3.6 不新增 Aurex 语言语法。它把 M3.0-M3.5 已稳定下来的 module、generic、query-backed sema、tooling session
和 stable syntax identity 提升到工程级执行模型：同一份 project/workspace 输入应同时驱动 CLI `--check`、
incremental cache、query graph 和 tooling session，而不是让 driver、module loader 和 IDE 各自临时拼装输入。

参考的现代编译器经验：

- Clang/LLVM 把 compilation database、target triple、driver options 和 module/dependency 文件作为可复现构建输入。
- rustc incremental compilation 把 crate graph、session options、dep graph node 和 on-disk query cache 分开建模。
- rust-analyzer 把 VFS/open files 和 crate graph 作为长期 workspace model，再把 query salsa-like facts 建在其上。
- Swift/Roslyn 维护 solution/workspace/project/document 分层，open buffer 是覆盖磁盘文件的显式输入。

Aurex 采用这些思想，但保持当前 Stage0 编译器的范围：只做 project model、project graph query、cache key 和
profile/invalidation 解释，不引入 package manager、dependency resolver、lockfile、registry protocol 或 version solver。

## 核心设计

### ProjectModel / WorkspaceModel

新增 `aurex_project` 目标，核心类型位于 `include/aurex/project/project_model.hpp`：

- `ProjectModelInput`：session kind、package root、source root、package identity、`PackageKey`、import roots、
  target config、command options 和 open buffers。
- `ProjectModel`：归一化后的单工程输入，包含稳定 `ProjectKey`、identity fingerprint 和 non-zero global id。
- `WorkspaceModel`：一组 project 的稳定 workspace identity。
- `ProjectTargetConfig`：emit kind、optimization level、clang path、clang args。
- `ProjectCommandOptions`：diagnostic format、query pruning 开关。
- `ProjectOpenBuffer`：path、uri、package identity、virtual buffer identity、source role、text fingerprint、text size
  和 generation。

归一化规则：

- package root、source root、import root 和 open buffer path 都走 canonical-or-absolute 路径。
- import roots 和 open buffers 排序后参与 identity，避免调用顺序影响稳定 key。
- open buffer text 通过 `project_open_buffer_fingerprint(...)` 进入 project identity。

### Driver / Tooling 适配层

driver 新增 `include/aurex/driver/project_model.hpp`：

- `project_model_from_invocation(...)`
- `workspace_model_from_invocation(...)`

CLI 输入会把 manifest root/source-root、`-I` import roots、package identity、emit kind、optimization level、
clang options、diagnostic format 和 query pruning 选项映射到 `ProjectModel`。

`ToolingSession` 持有 `project::WorkspaceModel`，并在 open/change/close document 时刷新：

- `ToolingProjectConfig` 新增 `source_root`。
- open buffer 使用 virtual source role、uri、package identity、buffer identity 和 text fingerprint 参与 project identity。
- tooling import root 使用 `tooling-import-root:` 前缀形成稳定 package identity。

### Query Graph

query 层新增：

- `QueryKind::project_graph`
- `ProjectKey`
- `project_graph_query_key(...)`
- `ProjectGraphProviderInput` / `ProjectGraphProviderOutput`
- `provide_project_graph_query(...)`
- `project_graph_query_record(...)`

`module_graph` 不再是没有依赖的孤立根节点。M3.6 后它显式依赖：

- 当前 project 的 `project_graph` query。
- 组成该 module 的 `module_part` query。

这样 module graph check 的调度从“module loader 递归加载顺序的副产物”变成 query graph 中可检查的显式节点。

### Persistent Query DB

incremental cache schema 从 1 升到 2，并在 header 中记录工程输入：

- `project_identity`
- `package_root`
- `source_root`
- `target_config`
- `command_options`
- `open_buffers`

cache query row 中新增 `project_graph` kind。query execution stats、query pruning profile、provider evaluation profile
和 subject ordering 都加入 project graph 计数。`module_graph` provider 的 dependency list 会写入并验证
`module_graph -> project_graph` 与 `module_graph -> module_part` 边。

### Invalidation / Profile

`try_reuse_incremental_check_cache(...)` 在 source fingerprint / query pruning 判断前先比较 project inputs。
profile 子事件新增：

- `incremental_cache.project_inputs`

detail 形状：

```text
result=reuse,changed_inputs=none
result=reject,changed_inputs=schema|project_identity|package_root|source_root|target_config|command_options|open_buffers
```

可解释的变化项包括 schema、compiler、mode、root、project identity、package root、source root、target config、
command options、open buffers、package、import paths 和 import packages。

## Work Packages

### WP-1 Project / Workspace Model

完成：

- 新增 `aurex_project` CMake target。
- 新增 `ProjectModel`、`WorkspaceModel`、target config、command options、import root、open buffer value types。
- 为 project/workspace 增加 stable serialize、debug string 和 stable key/fingerprint。

### WP-2 Driver / Tooling Shared Input

完成：

- driver invocation 到 project/workspace model 的 adapter。
- tooling session 持久持有 workspace model。
- open/change/close document 更新 open-buffer identity。
- tooling source root、relative path 和 import roots 进入同一套 project model 归一化规则。

### WP-3 Project Graph Query

完成：

- 新增 `ProjectKey` / `QueryKind::project_graph` / project graph provider。
- stable key decoder 支持 project key layout。
- query executor、context、provider set 和 result record 支持 project graph。
- module graph provider 接收显式 dependencies。

### WP-4 Cache Header / Invalidation / Profile

完成：

- incremental cache schema v2 写入 project inputs。
- cache parser、header match、validation 和 changed-input explanation 支持新字段。
- query stats、query pruning summary 和 provider-eval summary 支持 project graph。
- `PipelineProfileSubeventId::incremental_cache_project_inputs` 挂回 `incremental_cache.lookup`。

### WP-5 Tests / Docs / Gates

完成：

- query 单测覆盖 `ProjectKey`、stable layout、project graph provider、query executor 和 edge verifier。
- driver incremental-cache 回归覆盖 schema v2、project input profile、project graph row/edge、query pruning stats
  和 malformed graph repair。
- tooling session 测试覆盖 workspace model 在 open/change/close document 时的 open buffer identity 变化。
- M3.6 文档和路线状态已收口。

## 非目标

M3.6 明确不实现：

- package manager。
- dependency resolver。
- lockfile。
- registry protocol。
- version solver。
- 多线程 scheduler。
- 物理 syntax subtree graft。
- IDE completion/rename/semantic tokens。这些进入 M3.7。

## 验收标准

M3.6 的验收面：

- CLI `--check` 和 tooling session 消费同一套 project/workspace input shape。
- source root、package identity、target config、driver options、module graph 和 open buffers 会影响稳定 cache key。
- persistent cache 能写入/读取 `project_graph` query rows 和依赖边。
- module graph 依赖 project graph 和 module parts，且 query edge verifier 能拒绝 malformed kind/identity。
- profile 能解释 project input reuse/reject 和 changed inputs。
- 原有测试、coverage、query pruning、query graph fuzz 和 stress gates 保持 green。

## 后续

M3.7 可以在这个基础上推进 IDE semantic features：completion、rename、semantic tokens、code actions、
workspace symbols 和 stale-generation handling。M3.8 再把 query-backed reuse 推进到 lowering、IR 和 backend。
