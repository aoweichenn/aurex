# Aurex M3.8 Query-backed Lowering / Backend Reuse 计划与收口记录

## 状态

M3.8 已收口。该阶段把 M3.2-M3.4 的 query-backed sema authority 继续下沉到 lowering、IR pass pipeline 和 backend emission 边界。

## 设计原则

- `lower_function_ir` 不再由 sema/type-check fingerprint 伪造结果；只有真实完成 lowering 和 IR pass pipeline 后，incremental cache 才写入 lower IR query rows。
- 单个普通函数体和单个 generic function instance 都以 stable `BodyKey` / `GenericInstanceKey` 作为 query identity。
- target-independent IR unit、layout/ABI facts 和 LLVM emission unit 是三个不同 fingerprint 边界；backend 可以观测 LLVM unit 摘要，但 IDE/tooling 不直接消费 backend DTO。
- IR pass manager summary 显式暴露 scheduled/executed pass、changed 状态、preserved analyses 和 invalidated analyses。
- `check` / `typed` / `checked` emit mode 仍停在 sema/checked 输出，不写 lower IR rows；`ir` / `llvm-ir` / native emit mode 在 lowering/pass pipeline 后写包含真实 IR unit 的 incremental cache。

## 完成交付

1. Function body lowering query authority
   - `lower_function_ir` row 的 result fingerprint 现在混入 type-check result、真实 function IR unit fingerprint、LLVM emission unit fingerprint。
   - dependency edge 仍指向对应 `type_check_body`，但 result 对 dependency change 也敏感，避免 dependency 变化而 dependent row 被误判 reusable。

2. Generic instance lowering query authority
   - generic function instance body 通过 `GenericInstanceKey` 映射到 lowered IR function symbol。
   - `lower_function_ir` stable key 继续兼容 `GenericInstanceKey` 布局，dependency edge 指向 `generic_instance_body`。

3. Type layout / enum layout / ABI symbol facts
   - 新增 `ir::layout_abi_fingerprint(...)`，覆盖 record layout、payload enum record layout、global constants 和 function ABI signatures。
   - function IR unit fingerprint 混入 layout/ABI fingerprint，因此 layout、enum payload storage 或 ABI symbol 变化会使相关 lower IR/backend unit 不再复用旧事实。

4. IR pass preservation / invalidation
   - `PassPipelineRunSummary` 新增 `invalidated_analyses`。
   - `analysis_id_name(...)` 作为稳定 profile/test/debug 名称入口。
   - `LoweringPipeline` 的 `ir.pass_pipeline` profile detail 记录 scheduled/executed/changed/preserved/invalidated 摘要。

5. LLVM emission unit boundary
   - 新增 `ir::llvm_emission_unit_fingerprint(...)` 与 `ir::function_ir_unit_fingerprints(...)`。
   - `BackendPipeline` 的 `llvm.emit_ir` profile detail 记录 function unit 数量和 layout/ABI fingerprint global id。
   - backend 仍消费 optimized Aurex IR，不把 LLVM text 反向变成 query key。

## 测试覆盖

- `tests/gtest/ir/ir_fingerprint_tests.cpp` 覆盖 function IR unit、layout/ABI 和 LLVM emission unit 指纹分离。
- `tests/gtest/ir/pass_pipeline_tests.cpp` 覆盖 analysis name、preserved 和 invalidated analysis summary。
- `tests/gtest/driver/cli_driver_tests.cpp` 覆盖 typed emit 不写 lower IR rows、IR emit 写 lower IR rows、O0/O1 optimized IR unit fingerprint 变化、query schedule order、dependency edges，以及各 emit stop point 的 incremental cache 写失败传播。
